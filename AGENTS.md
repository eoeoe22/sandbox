# 이 프로젝트는 웹 브라우저 기반 샌드박스 파티클 시뮬레이터임.
1순위 : 재미
2순위 : 편의성
3순위 : 과학적 고증

# 문서 정책

- **Cloudwiki "Web sandbox 프로젝트" 문서 = 소개문서.** 개발 기록을 쌓는 곳이
  아니라 프로젝트를 처음 보는 사람에게 소개하는 가벼운 문서로 유지한다.
  개요·핵심 재미·주요 기능 하이라이트·플레이 가이드/배포 링크 정도만 담는다.
- **개발 기록·상세 레퍼런스 = 저장소 `docs/` 폴더.** 분야별로 파일을 나눠
  하나가 지나치게 커지지 않게 관리한다(`ARCHITECTURE.md`, `FEATURES.md`,
  `PHYSICS.md`, `MATERIALS.md`, `OBJECTS.md` 등). 새 분야는 새 파일 +
  `docs/README.md` 인덱스 링크로 추가한다.

# 워크플로우

1. 새 브랜치에서 작업 수행
2. 작업 후 1차 Push 및 gh CLI로 Pull Request 생성
3. 서브에이전트 호출을 통해 코드리뷰
4. 리뷰가 no issues로 통과 시 call_user 도구 사용 / 리뷰에서 지적사항이 있다면 조치후 리뷰 재시도 (no issues 확정 까지 반복)
5. 사용자의 승인 대기 (Merge는 사용자가 진행)
6. 개발 기록은 `docs/` 폴더의 해당 분야 문서를 갱신한다. Cloudwiki
   "Web sandbox 프로젝트" 소개문서는 하이라이트만 최신화(즉시 반영 편집 권장).
