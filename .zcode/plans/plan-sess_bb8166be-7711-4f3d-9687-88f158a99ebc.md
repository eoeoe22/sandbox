### 작업 1: 영역 선택(rect)에 직전 활성 도구 적용

**핵심 아이디어**: `$tool`이 'rect'일 때, rect는 **직전에 활성화된 비-rect 도구**(`$lastTool`)를 사각형에 적용. 좌클릭=해당 도구의 긍정 동작, 우클릭=지우기(기존 유지).

#### 1-1. `src/state/store.ts`
- `let $lastTool = atom<Tool>('material')` 추가 — rect가 아닌 도구로 전환될 때마다 갱신.
- `resetSettings()`에 `$lastTool.set('material')` 추가.

#### 1-2. 도구 전환 시 `$lastTool` 갱신
- `ControlPanel.svelte`에서 tool을 set하는 곳이 많으므로, 중앙 헬퍼 함수 `setTool(t)`를 만들어: `tool.set(t)` + `t !== 'rect'`면 `lastTool.set(t)`.
- 단, rect로 전환할 때는 lastTool을 건드리지 않음(직전 도구 보존).

#### 1-3. `PointerPainter.ts` — rect 분기에 도구별 적용
- `onDown`의 rect 분기에서 `rectErase` 외에 `rectTool = $lastTool.get()` 기록 (드래그 시작 시점 스냅샷).
- `paintRect`를 **도구별 분기**로 재구성:
  - **material/blend 직전 도구 + 좌클릭**: 기존대로 단일 재료 또는 `paintBlend`를 rect에 1회 적용.
  - **mix 직전 도구 + 좌클릭**: rect 셀 리스트를 만들어 `mixCells` 1회 호출.
  - **heat/cool 직전 도구 + 좉클릭**: rect에 가열/냉각 즉시 적용(작업 2의 강도 사용).
  - **우클릭(erase)**: 기존대로 Empty 채우기 + 오브젝트 삭제.
- rect 마퀴 오버레이 색상을 직전 도구에 맞게 표시(material=채우기, erase=지우기, heat=따뜻, cool=차가움, mix=보라, blend=혼합) — `dataset.mode`에 도구명 반영.

#### 1-4. rect용 셀 리스트 헬퍼
- `rectCells(sx,sy,ex,ey): number[]` — AABB의 모든 셀을 `[x,y,...]` 평배열로 반환(brushCells와 동일 포맷). heatCells/mixCells에 전달.

### 작업 2: 가열/냉각 도구 모드 + 강도 커스터마이징

#### 2-1. `src/game/config.ts` — 상수 추가
```
HEAT_BRUSH_MODE_DEFAULT = 'absolute'   // 'absolute' | 'percent'
HEAT_RATE_DEFAULT = 300                 // 절대 모드: 1초당 ±N도 (기존 12/stamp × 30Hz ≈ 360과 비슷한 규모)
HEAT_RATE_MIN = 10, HEAT_RATE_MAX = 2000, HEAT_RATE_STEP = 10
HEAT_PERCENT_DEFAULT = 100              // 퍼센트 모드: 1초당 ±N% (현재 온도 기준)
HEAT_PERCENT_MIN = 1, HEAT_PERCENT_MAX = 500, HEAT_PERCENT_STEP = 1
```

#### 2-2. `src/state/store.ts`
- 타입 `HeatMode = 'absolute' | 'percent'` + atom `$heatMode`, `$heatRate`, `$heatPercent`.
- `resetSettings()`에 기본값 추가.

#### 2-3. `src/state/persistence.ts` — 영속화 (4곳)
- `hydrateSettings`: `oneOf`/`clampFloat`로 검증 후 set.
- `saveSettings`: 3 필드 추가.
- `initSettingsPersistence`: 3개 `.listen(schedule)` 등록.

#### 2-4. `src/game/engine/brushTools.ts` — 퍼센트 모드 지원
- `heatCells`에 모드 파라미터 추가(또는 별도 함수):
  - **absolute**: `delta` = `(heatRate / stampsPerSecond)` per stamp. stampsPerSecond는 ×1=30, ×2=60 등 시뮬레이션 속도에 비례.
  - **percent**: `delta` = `temp * (percent/100 / stampsPerSecond)` per stamp — 현재 온도의 비율로 증감.
- 단순화: `stamp` 호출 시점에 delta를 계산해 `heatCells`에 전달(기존 ±12 고정을 동적 delta로 교체).

#### 2-5. `PointerPainter.ts` — stamp에서 delta 동적 계산
- `stamp`의 heat/cool 케이스에서 모드에 따라 delta 계산:
  - absolute: `heatRate / stampsPerSecond` (가열은 +, 냉각은 −)
  - percent: per-cell이므로 heatCells 내부에서 처리(각 셀의 현재 온도 × 비율). brushTools에 모드 전달.
- `stampsPerSecond`: 시뮬레이션 속도에 비례. TICK_HZ(60) × simSpeed 효과 / 2 (현재 stepMs 공식과 정합). 실제로는 `TICK_HZ * simSpeed / 2` (×1→30, ×2→60).

#### 2-6. 영역 선택(rect)에서의 가열/냉각 — "1초당 상승/하강 온도/퍼센트 즉시 적용"
- rect 가열/냉각 확정 시: 모드 무관 **1초치 분량을 한 번에** 적용.
  - absolute: rect 전체 셀에 `heatRate` (1초분량)를 ±적용.
  - percent: 각 셀에 `temp * (percent/100)`를 ±적용.
- 즉 `paintRect`의 heat/cool 분기에서 1초분량 delta 사용.

#### 2-7. `ControlPanel.svelte` — 설정 모달에 가열/냉각 필드 추가
- `frequentSettings` snippet에 `.field` 블록 추가:
  - 모드 세그먼트: 절대(`bi-thermometer`)/퍼센트(`bi-percent`)
  - 슬라이더: 모드에 따라 `heatRate`(10–2000, 단위 "도/초") 또는 `heatPercent`(1–500, 단위 "%/초") 토글 표시
- 기존 세그먼트/슬라이더 마크업 패턴 그대로 사용.

### 호환성 메모
- 기존 HEAT_BRUSH_DELTA(12) 상수는 폐기 또는 폴백용 유지. 기본 heatRate(300)로 기존 체감(약 360/초)과 유사하게 튜닝.
- rect의 좌클릭이 직전 도구를 따라가므로, 단순 재료 채우기는 "재료 도구 → 영역 도구" 순서로 기존과 동일한 워크플로우 유지.
- 우클릭은 항상 지우기(변경 없음).
